                                        Dave Ceddia Mod Stephen P 4 months ago

                                        Good question! I was thinking in terms of keeping everything in the component here, so I put it in the constructor (another spot to consider would be componentDidMount).

                                        In this case since those sounds aren't really instance-specific, you could create them outside the class and it would work fine. Anything instance-specific should go in the component somewhere though (because JS modules are singletons, and every `import` of a module would get the same vars otherwise).
